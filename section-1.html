Subsubsection 1.1.2.1 Programing

There is a somewhat energetic debate among the Data Science community about whether the subject should be taught using Python, R, or some other language. I have made the decision to teach it using Python for the following reasons:

  • Python is widely adopted across a large portion of the sciences,

  • Python is used extensively in industry for both Data Science and for other computations,

  • As measured by Google Searches, Python is an order of magnitude more used than R,

  • Python has an extensive collection of libraries for heavy machine learning and is the language used for interacting with cloud computing services like Google Cloud, Amazon Web Services and other heavy iron,

  • Python is much closer to having parallel programing for Data Science ready,

  • It is my assertion that after learning Data Science with Python, you will be able to quickly learn how to handle data in R (until you reach the limit of available packages).

However. As I said, there is still a debate about this. I am ultimately making the decision because my training in, and use of Data Science has been more extensive with Python. In fact Python is quickly replacing Spreadsheets, Matlab, and to some extent Mathematica in my daily work.

While we will be using Python, no prior knowledge of Python or programing is needed for the class. We will use examples to get started in all cases. In fact the class will serve as a gentle introduction to Python and programing.

That said, if you already know Python, you should use the class to expand your skill set. Remember the core principals of Python:

I suggest installing Jupyter and Python via the Anaconda distribution at this link . Subsection 1.1.4 Learning Objectives / Outcomes for the Course

As a result of this course, students will expand their skill set to work and think with data. They will

  1. be able to organize complex and messy datasets for analysis,

  2. develop prediction models,

  3. evalute the performance of prediction models,

  4. be able to communicate their analysis using dynamic and reproducible reports, and

  5. gain extensive experience using software for analysis.

Subsubsection 1.1.8.5 Packages and Writing Algorithms

It is a common question with this class: Do we need to write code to implement this algorithm or method? The answers to this question depend:

  1. For the Team Project you may use any Python module - most of what we do will be in Scikitlearn.

  2. At some point during class your team will need to write a code implementing one of our mehtods. This will be one of the subassignments for the project.

  3. You will need to write some of your own error checking methods to evaluate methods as part of the project - particulcarly if the standard error estimate is not the best for your problem.

  4. The Team Activities will involve more specific algorithm implementation.

The difficulty for you, as a Data Scientists is:

  • All of the methods we will use are basically already coded as part of scikitlearn, unless you want to build your own Ensemble or other more involved model,

  • However if you do not understand what a method is doing you will not be aware of some of the ways it can fail or lead you to false conclusions.

This last point can be mitigated by understanding the Data Science Process and following the procedures to get accurate estimates on the error of your model. I.e. it is not necessary to always understand exactly what a model is doing (for example, we generally do not really know why Neural Networks work other than a heuristic for why they should work)

  • however if we have been true to the error estimating process we should be able to quanitfy how well the model did.

That said. Sometimes we will be wrong. The more critical a model is to our company, the more important it is that we control the process and account for possible errors. I.e.

  • When I'm goofing off on Kaggle learning new methods, I sequester just one subset of the data,

  • If we were developing a model the city would use to determine which water lines need maitenance we would need to be sure we understand the chance of false negatives from our model and have multiple levels of sequestered data, and

  • If we were developing a model the state would use to decide which inmates up for parole were likely to reoffend we would absolutely want to know the errors our model had, isolated multiple levels of data in the building process would be essential.

Subsubsection 1.1.8.7 Grading Policies

Your grade in the class will be based on homework, team activities, the team project, and class presentations.

The weighted distribution of assessments towards your final grade will be:

  • Homework: 10%

  • Team Activities: 10%

  • Team Project: 70%

  • Class Presentations: 10%

The Team Project will be assessed throughout the semester and will include the many small assignments that will eventually add up to the whole project.

Final Grades will be based on a percentage of this weighted total. The distribution will be no tougher than A 90

  • 100%, B 80 - 89%, C 70 - 79%, D 60 - 69%, and F 1 - 59%.

Subsubsection 1.1.8.8 Grading and Feedback

I will be providing you feedback on the Homework, Team Activities, Class Presentations, and during the semester the Team Project's sub-assignments.

At the end of the semester the Team Projects will be assessed by me, and by the class. The final grade on them will be computed by a weighted combination of all scores.
